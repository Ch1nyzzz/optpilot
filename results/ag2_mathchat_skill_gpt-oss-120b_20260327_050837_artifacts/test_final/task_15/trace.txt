Task: From an unlimited supply of 1-cent coins, 10-cent coins, and 25-cent coins, Silas wants to find a collection of coins that has a total value of $ N $ cents, where $ N $ is a positive integer. He uses the so-called **greedy algorithm**, successively choosing the coin of greatest value that does not cause the value of his collection to exceed $ N $. For example, to get 42 cents, Silas will choose a 25-cent coin, then a 10-cent coin, then 7 1-cent coins. However, this collection of 9 coins uses more coins than necessary to get a total of 42 cents; indeed, choosing 4 10-cent coins and 2 1-cent coins achieves the same total value with only 6 coins.

In general, the greedy algorithm succeeds for a given $ N $ if no other collection of 1-cent, 10-cent, and 25-cent coins gives a total value of $ N $ cents using strictly fewer coins than the collection given by the greedy algorithm. Find the number of values of $ N $ between 1 and 1000 inclusive for which the greedy algorithm succeeds.

We have three coin denominations:  

\[
1,\;10,\;25\text{ (cents)}
\]

For a given amount \(N\) ( \(1\le N\le 1000\) ) Silas applies the *greedy algorithm*:  
take as many 25‑cent coins as possible, then as many 10‑cent coins as possible, and finally fill the rest with 1‑cent coins.

The greedy algorithm *succeeds* for \(N\) if the number of coins it uses is **minimum** among all possible collections of the three denominations.

The task is to count how many values of \(N\) in \([1,1000]\) satisfy this property.

--------------------------------------------------------------------

### 1.  Formalising the two counts  

*Greedy count* \(G(N)\) – obtained directly from the greedy rule:

```
g = N // 25          # number of 25‑cent coins
N1 = N - 25*g
t = N1 // 10         # number of 10‑cent coins
N2 = N1 - 10*t
o = N2               # number of 1‑cent coins
G(N) = g + t + o
```

*Optimal count* \(O(N)\) – the true minimum number of coins needed for \(N\).  
Because the coin set is small, a simple dynamic‑programming (DP) table works:

```
dp[0] = 0
dp[x] = 1 + min(dp[x-1], dp[x-10] (if x≥10), dp[x-25] (if x≥25))
```

After filling the table up to 1000, \(O(N)=dp[N]\).
