### 1.  What has to be erased?

The original equality  

\[
\underbrace{(x-1)(x-2)\cdots (x-2016)}_{\text{left side}}
\;=\;
\underbrace{(x-1)(x-2)\cdots (x-2016)}_{\text{right side}}
\tag{1}
\]

contains the same linear factor \((x-k)\) on **both** sides for every
\(k\in\{1,2,\dots ,2016\}\).

If after erasing we keep the factor \((x-k)\) on the two sides,
then at the point \(x=k\) both sides become \(0\); consequently
\(x=k\) is a real root of the new equation.
Hence, in order that the final equation have **no** real root,
**no integer \(k\) may stay on both sides**.

Consequently each integer \(k\) has to be erased from at least one
of the two sides.  
If we erase it from exactly one side, the total number of erased
factors is

\[
\underbrace{2016}_{\text{one copy of each integer}}
\]

and we keep every integer on exactly one side.  
Thus the problem reduces to the following:

> Find a partition of the set \(\{1,2,\dots ,2016\}\) into two non‑empty
> subsets \(A\) and \(B\) such that the equation  

\[
\prod_{i\in A}(x-i)=\prod_{j\in B}(x-j) \tag{2}
\]

has **no real solution**.

If such a partition exists, the required minimum number of erased
factors is exactly \(2016\); if it does not exist we would have to
erase more factors.

--------------------------------------------------------------------

### 2.  A partition with no real solution

Take  

\[
A=\{2016\},\qquad 
B=\{1,2,\dots ,2015\}.
\]

Equation (2) becomes  

\[
x-2016=(x-1)(x-2)\cdots (x-2015). \tag{3}
\]

Define  

\[
F(x)=\prod_{j=1}^{2015}(x-j)-(x-2016).
\]

* For \(x\le 1\) every factor \((x-j)\) is \(\le 0\); the product on the
  left‑hand side is therefore **non‑negative**, while the term
  \((x-2016)\) is negative. Hence \(F(x)>0\) for all \(x\le 1\).

* For \(x\ge 2016\) each factor \((x-j)\) is positive, so the product on
  the left‑hand side is positive and dominates the linear term

import itertools, sympy as sp

def exists_n(n, max_val=12):
    xs = sp.symbols('x')
    numbers = list(range(1, max_val+1))
    for A in itertools.combinations(numbers, n):
        for B in itertools.combinations(numbers, n):
            if set(A)&set(B): # allow overlap? Overlap leads to common roots => equality at those roots, not allowed.
                continue
            f = sp.prod([xs - a for a in A])
            g = sp.prod([xs - b for b in B])
            h = sp.expand(f - g)
            # check if h has any real root
            # compute real roots numerically
            roots = sp.nroots(h)
            real = any(abs(r.as_real_imag()[1])<1e-8 for r in roots)
            if not real:
                return A,B
    return None

for n in [1,3,5]:

    print(n, res)
